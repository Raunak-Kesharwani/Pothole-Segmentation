# Contributing to Pothole Segmentation

We welcome contributions to make Indian roads safer!

## Code of Conduct
Please be respectful and professional in all communications.

## How to Contribute

1.  **Fork the repository**.
2.  **Create a feature branch**:
    ```bash
    git checkout -b feature/amazing-feature
    ```
3.  **Commit your changes**:
    ```bash
    git commit -m 'Add some amazing feature'
    ```
4.  **Push to the branch**:
    ```bash
    git push origin feature/amazing-feature
    ```
5.  **Open a Pull Request**.

## Coding Standards

-   **TypeScript**: Use interfaces for all data structures.
-   **Styling**: Use Tailwind utility classes. Avoid inline styles.
-   **Components**: Keep components small and focused. Use `c:/Users/musaq/Work/Pothole-Segmentation/frontend/src/components/ui` for basic elements.
-   **Commits**: Use semantic commit messages (e.g., `feat:`, `fix:`, `docs:`, `chore:`).

## Reporting Bugs

Please check existing issues before reporting. Include:
-   Device/Browser version.
-   Steps to reproduce.
-   Screenshots if applicable.
